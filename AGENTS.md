# AGENTS.md

このファイルは、このリポジトリで Codex や他の AI エージェントに作業を依頼するときの共通ルールです。

## Project Overview

- このプロジェクトは Nuxt 4 + Nuxt Content を使った個人技術ブログです。
- 目標は、技術記事・読了本・プロフィール・アーカイブを見やすくまとめたブログサイトを作ることです。
- UI は Nuxt UI を土台にし、白基調のカード型レイアウト、タグ、検索、ダークモードに対応する方針です。

## Tech Stack

- Framework: Nuxt 4
- Content: Nuxt Content
- UI: Nuxt UI を採用予定
- Package manager: pnpm
- Preferred command runner: Vite Plus
- Language: TypeScript / Vue

## Commands

基本的には Vite Plus のコマンドを優先してください。
対応する Vite Plus コマンドが不明な場合や、依存関係管理などで必要な場合のみ pnpm を使ってください。

```bash
pnpm install
pnpm dev
pnpm build
pnpm generate
pnpm preview
```

## Development Guidelines

- 既存の Nuxt / Vue / Nuxt Content の構成を尊重してください。
- 大きな変更をする前に、関連ファイルを読んでから作業してください。
- UI 実装では Nuxt UI のコンポーネントを優先して使ってください。
- コンポーネントは必要以上に複雑にせず、まず読みやすさを優先してください。
- 記事や本のデータは、可能な限り Markdown frontmatter で管理してください。
- CSS を追加する場合は、局所的な調整にとどめ、共通化できるものはテーマや共通クラスに寄せてください。

## Content Guidelines

- 技術記事は `content/` 配下で管理します。
- 記事データには、タイトル、説明、日付、タグ、サムネイル、読了時間を持たせる想定です。
- 読了本データには、タイトル、著者、読了日、評価、カテゴリ、表紙画像を持たせる想定です。
- 実装時点でディレクトリ構成が決まっていない場合は、まず構成案を提示してから進めてください。

## UI Direction

- 添付デザインのような、技術ブログらしいクリーンな見た目を目指します。
- トップページには、記事一覧、読了本一覧、プロフィール、サイト説明、アーカイブを配置します。
- PC ではメインカラム + サイドバー、モバイルでは 1 カラムにしてください。
- 余白、角丸、影は控えめにし、情報を探しやすい密度にしてください。
- ダークモード対応を前提に、色を直接固定しすぎないでください。

## Safety Rules

- ユーザーが明示していない既存変更を勝手に戻さないでください。
- 破壊的な Git 操作は行わないでください。
- 依存関係の追加は、必要性を説明してから行ってください。
- 実装後は可能な範囲で Vite Plus のビルド/検証コマンドを使って確認してください。
- Vite Plus の対応コマンドが分からない場合は、pnpm scripts をフォールバックとして使ってください。
